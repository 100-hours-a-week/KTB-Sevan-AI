# 🐱 LinkCat: URL 분류 및 관리 프로그램

**LinkCat**은 터미널 환경에서 웹사이트 링크(URL)를 카테고리별로 분류하여 저장하고, 필요할 때마다 카테고리별로 빠르게 조회할 수 있는 파이썬 기반의 CLI(Command Line Interface) 북마크 관리 도구입니다.

---

## 주요 기능

* **카테고리별 링크 저장 (`add`):** 유저가 입력한 URL과 카테고리를 매핑하여 저장합니다. 동일한 카테고리 내 중복 URL 저장을 방지하는 안전장치가 포함되어 있습니다.
* **유연한 목록 조회 (`list`):** 카테고리를 지정하지 않으면 저장된 전체 목록을 카테고리별 등록 개수와 함께 출력하며, 특정 카테고리를 지정하면 해당 카테고리의 링크들만 필터링하여 보여줍니다.
* **JSON 기반 데이터 영속성:** 별도의 데이터베이스 서버 설치 없이 로컬의 `links.json` 파일 하나로 데이터를 관리하여 이식성과 가벼움을 극대화했습니다.

---

## 사용 기술

* **Language:** Python 3.11+
* **CLI Arguments Parsing:** `argparse` (Python 내장 모듈)
* **Data Storage & I/O:** `json`, `os` (Python 내장 모듈)

---

## 프로젝트 구조

```text
.
├── main.py       # 인터페이스 레이어 (CLI 명령어 라우팅 및 인자 파싱)
├── manager.py    # 데이터 레이어 (JSON 파일 생성, 로드, 데이터 저장 및 출력 로직)
└── links.json    # 영속성을 보장하는 키-값(Key-Value) 구조의 로컬 데이터베이스

```

## 설치 및 실행 방법

### 1. 가상환경 생성 및 활성화

```bash
# 가상환경 생성
python3 -m venv venv

# 가상환경 활성화 (Mac/Linux)
source venv/bin/activate

# 가상환경 활성화 (Windows CMD)
# venv\Scripts\activate.bat

```

### 2. 프로그램 실행 규칙

본 프로그램은 별도의 외부 패키지 설치 없이 파이썬 내장 모듈만으로 구동되므로 즉시 실행이 가능합니다.

```bash
# 링크 추가 실행 구조
python main.py add -u <URL주소> -c <카테고리명>

# 목록 조회 실행 구조
python main.py list [-c <카테고리명>]

```

## 💻 실행 예시

### 1. 새로운 링크 추가 (`add`)

```bash
python main.py add -u "https://docs.python.org/3/" -c "개발_공식문서"

```

*(성공적으로 반영되면 로컬에 `links.json` 파일이 자동 생성되거나 업데이트됩니다.)*

### 2. 전체 목록 조회 (`list` 카테고리 미지정)

```bash
python main.py list

```

**출력 결과:**

```text
전체 목록을 보여드립니다.
========================================
개발_공식문서(1 개)
  - https://docs.python.org/3/
인공지능(2 개)
  - https://chatgpt.com
  - https://gemini.google.com

```

### 3. 특정 카테고리 필터링 조회 (`list -c`)

```bash
python main.py list -c "인공지능"

```

**출력 결과:**

```text
인공지능(2 개)
  - https://chatgpt.com
  - https://gemini.google.com

```

## 📝 회고 및 배운점

### 🎯 주제 선정 및 기초 구현

* **주제 선정 배경:** 일상적이고 반복적인 개발 공부 과정에서 유용한 웹사이트 링크나 공식 문서 주소들이 브라우저 탭이나 메모장에 파편화되는 문제를 겪었습니다. 이를 터미널 환경에서 직관적이고 구조적으로 관리할 수 있는 '나만의 유틸리티 도구'를 만들고자 본 프로젝트를 기획했습니다.
* **기초 구현 방식:** 사용자가 명령어 환경에 익숙해질 수 있도록 `argparse` 모듈의 `subparsers` 기능을 활용했습니다. 명령어의 성격에 따라 `add`와 `list`로 라우팅을 명확히 분리하여 뼈대를 잡았습니다.

### 💡 깨달은 점 및 배운점

* **관심사의 분리 (Separation of Concerns):** 사용자의 입력을 파싱하는 인터페이스 레이어(`main.py`)와 파일에 접근하여 데이터를 핸들링하는 비즈니스/데이터 레이어(`manager.py`)를 엄격히 분리하여 구현했습니다. 이 과정에서 코드의 유지보수성이 왜 향상되는지, 결합도가 낮아진다는 것이 무엇인지 몸소 체감할 수 있었습니다.
* **데이터 구조화와 예외 처리:** 파이썬의 딕셔너리 자료형과 JSON의 데이터 형식이 1:1로 유기적으로 매핑되는 구조를 학습했습니다. 특정 카테고리가 존재하지 않을 때 발생하는 오류를 방지하기 위해 `if category not in db:` 같은 방어적 코드(안전장치)를 적재적소에 배치하는 흐름의 중요성을 배웠습니다.
* **CLI 가독성의 가치:** 데이터를 가공하지 않고 딕셔너리 통째로 출력했을 때의 시각적 불편함을 겪은 후, `.items()`와 중첩 반복문(`for` 루프내 `for` 루프) 및 들여쓰기 공간 처리를 통해 터미널 환경에서의 가독성이 유저 경험(UX)에 얼마나 큰 영향을 미치는지 깨달았습니다.
